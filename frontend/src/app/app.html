Presentación del proyecto

Hola, Casa Mecate este es mi reto FullStack

Desarrollé una aplicación FullStack que integra un backend en Python con FastAPI, una base de datos SQL con SQLite, consumo de una API externa de StackOverflow, dockerización del backend y una interfaz en Angular para visualizar los resultados de forma clara, ordenada e intuitiva.

Ver documentación Swagger

Backend

FastAPI como intermediario

El backend recibe información desde dos orígenes: una API externa en formato JSON y una base de datos SQL local. Después procesa los datos, calcula resultados y expone endpoints REST para que Angular pueda consumirlos.

Frontend

Angular como vista principal

El frontend fue construido en Angular para presentar la información del reto mediante secciones, cards, tablas y bloques explicativos. La interfaz está pensada para que el flujo sea entendible tanto para revisión técnica como para revisión general.

Docker

Backend dockerizado

El backend se ejecuta mediante Docker usando Dockerfile y docker-compose. Esto permite levantar la API de forma consistente, aislada y fácil de revisar desde cualquier entorno compatible.

Arquitectura general

Flujo de la aplicación

La solución sigue el flujo solicitado en el reto: los datos SQL y JSON son procesados por el backend, y posteriormente el frontend consume los servicios REST para mostrar la información en pantalla.

Paso 1 SQL / JSON

SQLite almacena los datos de vuelos y StackExchange API entrega información externa en formato JSON.

Paso 2 Backend

FastAPI consulta, procesa, calcula y expone la información mediante endpoints REST.

Paso 3 Vista Angular

Angular consume los endpoints del backend y presenta los resultados de manera visual.

Resumen de módulos

Módulos desarrollados

La aplicación se divide en dos módulos principales. Cada módulo tiene su propia sección en el sidebar para revisar resultados, endpoints, respuestas e interpretación.
